IntelliJ IDEA 新手常用快捷键整理

2024-05-15 阅读时间: 约 5 分钟

大家好！在上一篇笔记中，我们成功运行了第一个 Java 程序。工欲善其事，必先利其器。今天，我想和大家分享一些我在使用 IntelliJ IDEA (简称 IDEA) 过程中收集和整理的常用快捷键。熟练使用这些快捷键，可以让你的编码速度和效率大大提升。

注意：以下快捷键以 Windows / Linux 系统为准。Mac 用户通常可以将 Ctrl 键替换为 Cmd 键。

1. 编辑操作 (最常用)

  • Ctrl + Space：代码自动补全。这是最最常用的快捷键，没有之一！在写代码时遇到不确定的类名、方法名，或者想快速输入 System.out.println() 时，都可以用它。
  • Ctrl + D：复制当前行或选中的代码块到下一行。非常方便地复制粘贴。
  • Ctrl + Y：删除当前行。比选中整行再按 Delete 键快得多。
  • Ctrl + /：单行注释 / 取消注释。选中多行时，会为每一行加上 //。
  • Ctrl + Shift + /：块注释 / 取消注释。会用 /* ... */ 将选中的代码块包裹起来。
  • Ctrl + Alt + L：格式化代码。让你的代码结构清晰，风格统一，强迫症的福音。

2. 代码导航

  • Ctrl + B 或 Ctrl + 鼠标左键点击：跳转到变量、方法或类的定义处。比如想看看一个方法是怎么实现的，用这个快捷键就对了。
  • Alt + ← / →：在不同的代码标签页或编辑位置之间切换。
  • Ctrl + Shift + F：在整个项目中搜索文本。想找一个变量或方法在哪里被使用了，这个功能非常强大。

3. 重构与优化

  • Shift + F6：重命名变量、方法、类等。这是一个非常安全的重构功能，它会自动更新所有引用到该名称的地方，避免了手动修改可能带来的遗漏。
  • Ctrl + Alt + O：优化导入。自动移除未使用的 import 语句，并按字母顺序排序。让你的代码顶部保持干净。

4. 运行与调试

  • Shift + F10：运行当前项目或测试。
  • Shift + F9：调试当前项目。
  • F8：在调试模式下，单步执行（Step Over），不进入方法内部。
  • F7：在调试模式下，单步执行（Step Into），进入方法内部。

5. 其他实用功能

  • Alt + Insert：生成代码。可以快速生成构造函数、getter/setter 方法、toString() 等。在类的空白处按下此快捷键，会弹出一个菜单供你选择。
  • Ctrl + Tab：在不同的文件和工具窗口之间切换。
  • Ctrl + F4：关闭当前的编辑标签页。

总结

以上就是我认为新手最应该掌握的一些 IDEA 快捷键。不必死记硬背，可以在日常编程中刻意去练习使用，慢慢就会形成肌肉记忆。IDE 还有很多其他快捷键，你可以通过 Help -> Keymap Reference 查看完整的列表。
